Tutorial-4: Javascript Part-1

HTML and CSS allow you to add visual elements to your page, and set their style properties.  You can also define some interactive behavior using CSS.  But, to make complex pages that update in response to user inputs or new data, we need Javascript, the primary language of the web.  We will use ES6 (6th version of the JS standard ECMAScript) which has significant upgrades compared to the previous version.  

We will focus initially on Javascript code that runs on the 'client side', updating visual elements on the page within the user's browser, rather than 'server-side' Javascript code.  

What does Javascript allow us to do?  Javascript can connect to the DOM.  You can access elements you created using html, to change or update them or 'listen' to the interactions users perform on these elements.  You can also add new elements to the DOM using Javascript, appending child elements to existing page elements or to the root.   

What is Javascript like?  For me, Javascript prizes flexibility and letting developers do what they want.  It is a bit like the wild west.  It can be written in several styles- procedural, object-oriented or functional.   You may have never worked with a functional language before, and it may be a bit confusing at first, but it can be very powerful!   Javascript is dynamically typed, which means that you can change a variable from string to int to an object anywhere in your code, without causing an error... See- Wild West! 

Variables:

As I mentioned above- Javascript variables are dynamically typed- you can switch a variable from an int to a string to an object, and no errors will be thrown.  Be careful with this feature though, because switching variable types can lead to bugs.  When I write code in Javascript, I don't typically find myself changing the type of my variables, because it seems like a bad idea.   

You also do not need to declare variables ahead of using them.  If it encounters a variable name that has not been declared, Javascript will just create a global variable for you.   Again-  convenient but a potential source of bugs.  This is a problem for me when I try to use a variable I declared previously, but I misspell it, and then Javascript thinks I am making a new variable.   Watch for this! 

For local variables (variables you use within a block of your code, like a function call), I recommend you declare variables in two ways. 

  • Use const for unchanging parameters
  • Use let whenever you need to reassign a variable.

You may see examples online where people use  'var' for variables that need to be reassigned,  and you could use var.  I have used var for lots of my Javascript code without thinking about it.   But 'let' is block scoped, so it can't be used outside of the block it is defined in, and it may allow you to avoid some issues.   More details here: https://www.freecodecamp.org/news/var-let-and-const-whats-the-difference/ (Links to an external site.)

//this is an example from:  https://github.com/UBC-InfoVis/2021-436V-tutorials/tree/master/0_Web_Tutorial (Links to an external site.)
// `const` keyword ensures that the variable it creates is read-only
const THRESHOLD = 10;

// `let` keyword declares a block-scoped variable
let month = "February";	

// Variables in JS are loosely typed
let day = 4;				// Integer
let temperature = 34.36; 	// Double
let winter = true; 			// Boolean

// `const` does not ensure that the variable is immutable
// the object property can change
const students = { 
	count: 10
};
students.count += 1; 		// OK: count=11
students = { count: 20 }; 	// Type error


To review:

  • No identifier creates a variable with global scope
  • let creates a variable with block scope
  • const creates a variable with block scope, and the variable cannot be reassigned.  
  • var creates a variable with function scope- which means that unless it is in a function it can be used again anywhere.  Not what you are probably used to. This is considered old form. 

So again- const and let are your best approaches for variable declaration. 

Arrays:

As in other languages, arrays can store a sequence of values, and contain any type of data- integers, strings objects.   The example below is going to look pretty similar to array operations in other languages.  

//this is an example from:  https://github.com/UBC-InfoVis/2021-436V-tutorials/tree/master/0_Web_Tutorial (Links to an external site.)
// Array with integer values
let numbers = [1, 2, 3, 100, 500, 4];	

// Array with strings
let fruits = ["Orange", "Banana", "Apple"];

// Empty array declaration
let names = [];

// Access the elements of an array
fruits[0]; 	// Returns: Orange
fruits[1]; 	// Returns: Banana

// Adding array elements
fruits.push("Mango");	// adds a new element to fruits

// Access the length of an array using the length attribute
let numberOfFruits = fruits.length;

// You can nest arrays (multidimensional)
let nestedNumbers = [[1, 2], [3, 4], [5, 6]];

But, on top of these basic operations, Javascript has many convenient functions for operating on arrays.  We are going to spend some time on these later in the course.  

Objects

Objects in javascript can store multiple variables.   You use curly brackets { } to define them, and set the properties and values.  You can add parameters to objects on the fly, not just at declaration.  

//this is an example from:  https://github.com/UBC-InfoVis/2021-436V-tutorials/tree/master/0_Web_Tutorial (Links to an external site.)
// JS object with five properties
let course = {
	id: "CS6067001",
	name: "User Interface Design",
	students: 30,
	active: true,
        students: ["Michael", "Ann", "James", "Kathy"] 
}

// Accessing an object via dot notation, specifying the name of the property
course.id; 		// Returns: CS6067001
course.students;	// Returns: 30

// We can add parameters to objects on the fly
course.schedule = "mwf"
course.time = "1:25pm"

// And we can also create arrays of objects
let courses = [
	{ id: "CS101", name: "Intro to CS" },
	{ id: "CS102", name: "Intro to CS 2" }
];

// To access this data we just follow the trail of properties
courses[1].id; 	// Returns: CS102

Keep in mind: [...] means array and {...} means object!

Control Structures & Loops


You should already be familiar with control structures, loops and functions. The following just shows some examples and the correct syntax for using those structures.


If statements



    
//this is an example from:  https://github.com/UBC-InfoVis/2021-436V-tutorials/tree/master/0_Web_Tutorial (Links to an external site.)
let numericData = 10;

// Regular if statement 
if (numericData >= 10) {
	console.log("Equal or greater than 10");
} else if (numericData > 0 && numericData < 10) {
	console.log("Between 0 and 10");
} else {
	console.log("Less than 1");
}

// The ternary operator can be used as a compact alternative to an if-statement
// CONDITION ? WHAT_HAPPENS_IF_TRUE : WHAT_HAPPENS_IF_FALSE
let result = (numericData >= 10) ? "greater than or equal to 10" : "less than 10";
let result2 = (numericData % 2 === 0) ? "even" : "odd";




For loops



    
//this is an example from:  https://github.com/UBC-InfoVis/2021-436V-tutorials/tree/master/0_Web_Tutorial (Links to an external site.)
// (1) Loop through a block of code 5 times (printing the value of i each time to the console)
for (let i = 0; i < 5; i++) {
	console.log(i);
}

// (2) Loop through each of the values in an array
let arrayWithNames = ["Jack", "Anna", "Mike", "Susan"];
for (let i = 0; i < arrayWithNames.length; i++) {
	console.log(arrayWithNames[i]);
}

// (3) Loop through the properties of an object
let person = { firstName: "John", lastName: "Doe", age: 24 };
for (let property in person) {
	console.log(property + ": " + person[property]);
}

// (4) The holy grail of JS loops: 
// Making use of object oriented JS, the forEach loop is an array method 
// that iterates over all elements in the array. The index of the element 
// and the element itself are available inside the loop via an anonymous
// callback function.
arrayWithNames.forEach( (element, index) => {
	console.log(index + ": " + element);
});




 


Template literals


You can concatenate strings and variables conveniently in Javascript using Template literals.


Here are some examples:  



    
const temp = 39; 
const message = `The expected max. temperature today is ${temp}°C`;





Template literals are specified using back-ticks (`), which distinguishes them from regular strings that are specified using single or double quotes.  You can use a ${expression} inside the literal.  You can do math in the expression, or call a function.  



    
// Function returns a dynamic variable that is embedded using template literals 
const message = `Current temperature: ${toCelsius(34)}°C`;




Template literals are a concise way to manipulate strings, and they can also span multiple lines. 


Functions- basics


There are lots of ways to make function calls in javascript- here are some basic ones.  We will go over more ways in the next tutorial, and how Javascript can be used in a functional programming paradigm.  But, at first, you can use functions in ways that are similar to other languages.

Accessing and modifying the DOM:


With Javascript you can access elements in the DOM- elements you have created using html, or elements you added in other places in your javascript code.  You can add new elements, you can access and modify attributes, including style settings.


You can access elements by their id:


let element = document.getElementById("intro");


You can access a list of elements by tag name, such as accessing all paragraph tags:


let element = document.getElementsByTagName("p");


You can access a list of elements by class name:


const x = document.getElementsByClassName("intro");
